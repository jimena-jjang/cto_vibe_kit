# /start-session — 세션 로그 자동 생성

> **⚠️ 모든 코드 작업 전에 이 워크플로우를 먼저 실행해야 합니다.**
> 사용자가 작업을 요청하면, 코드를 수정하기 **전에** 이 워크플로우를 실행하세요.

## 실행 조건

- 사용자가 코드 수정, 기능 추가, 버그 수정, 테스트 작성 등 **코드 변경을 수반하는 작업**을 요청한 경우
- 새로운 작업 세션이 시작될 때 (이전 작업과 다른 주제)
- 단순 질문이나 파일 읽기만 하는 경우는 **제외**

## Steps

// turbo
1. `docs/context.md` 파일의 첫 30줄을 읽어 현재 프로젝트 상태를 파악합니다.

```bash
head -30 docs/context.md
```

// turbo
2. 오늘 날짜로 된 세션 블록이 `docs/agent_session_log.md`에 이미 있는지 확인합니다.

```bash
TODAY=$(date +%Y-%m-%d)
if grep -q "$TODAY" docs/agent_session_log.md; then
  echo "✅ 오늘 날짜 세션이 이미 존재합니다. 기존 세션에 추가합니다."
else
  echo "📝 새 세션 블록을 생성해야 합니다."
fi
```

3. `docs/agent_session_log.md` 파일 끝에 새 세션 블록을 추가합니다.
   **반드시 아래 형식을 따르세요:**

```markdown
---

## {오늘 날짜} 세션: {작업 제목}

**참여**: 🎖️ PL, {관련 Sub-Agent들}

> **🎖️ PL**: {사용자 요청 요약 및 작업 목표 선언}

> **🎖️ PL → {Sub-Agent}**: {첫 번째 지시 사항}
```

4. 세션 블록이 정상 생성되었는지 확인합니다.

```bash
tail -10 docs/agent_session_log.md
```

5. **이제 코드 작업을 시작합니다.** 작업 중간에 중요한 결정이나 발견이 있으면 `docs/agent_session_log.md`에 추가합니다.

## 작업 중 기록 가이드

작업 도중 아래 상황이 발생하면 세션 로그에 추가 기록합니다:

| 상황 | 기록 형식 |
|------|----------|
| Sub-Agent에게 지시 | `> **🎖️ PL → {Agent}**: {지시}` |
| Sub-Agent 결과 보고 | `> **{Agent}**: {결과}` |
| 에러/이슈 발견 | `> **{Agent}**: ⚠️ {이슈 설명}` |
| 중요 결정 | `> **🎖️ PL**: {결정 내용}` |

## 작업 완료 시

커밋 전에 반드시 세션 결론을 추가합니다:

```markdown
### 세션 결론
- {Agent}: {완료 내용}
- {Agent}: {완료 내용}
- PL: {검수 결과}
```
